Rule Descriptions

Page Layout - H25 Title Tag

Description

  1. This method conforms to the WCAG rule H25.
  2. This rule identifies the title element in the head section that defines in a simple phrase the purpose of the document.
  3. This helps users to orient themselves within the site quickly without having to search for orientation information in the body of the page.

Procedure

  • Examine the source code of the HTML or XHTML document.
  • Check that a non-empty title element appears in the head section.

Page Layout - H57 HTML Tag Lang Attribute

Description

  1. This method conforms to the WCAG rule H57.
  2. This rule is to Identify the language of the document which is important for a number of reasons.
  3. This helps users to orient themselves within the site quickly without having to search for orientation information in the body of the page.
  4. Speech synthesizers that support multiple languages will be able to orient and adapt to the pronunciation and syntax that are specific to the language of the page, speaking the text in the appropriate accent with proper pronunciation.
  5. Marking the language can benefit future developments in technology, for example users who are unable to translate between languages themselves will be able to use machines to translate unfamiliar languages.

Procedure

  • Examine the html element of the document.
  • Check that the html element has a lang and/or xml:lang attribute.

Page Layout - H42 H1 Heading

Description

  1. This method conforms to the WCAG rule H42.
  2. This rule identifies the page has only one H1 tag associated with it.

Procedure

  • Check for the presence of h1 heading markup in the page.
  • Check that there are not multiple h1 tags.

Page Layout - H42 Heading Hierarchy

Description

  1. This method conforms to the WCAG rule H42.
  2. This rule identifies headings are used in a hierarchical layout with h3 as a subsection of h2, which is a subsection of h1.

Page Layout - Validate Skip to Main Content

Description

  1. This method conforms to the WCAG rule 2.4.1.
  2. This method checks whether a user for a screen reader is given provision to skip to the main content.
  3. This check verifies unsighted user if wishes to jump or not straight into the main content of the page.

Form - H44 Input Tag Label

Description

  1. This method conforms to the WCAG rule H44.
  2. This rule identifies for the presence of label attached to a specific form control through the use of the for attribute.

Procedure

  • List1 of all the input elements id is identified and collected.
  • List2 of all the label for attribute value is identified and collected.
  • Comparing the two lists for the presence of List1 item through iteration is present in the List2.

Form - H32 Form Submit Button

Description

  1. This method conforms to the WCAG rule H32.
  2. This rule identifies the mechanism that allows users to explicitly request changes of context.

Procedure

  • Find all forms in the content and for each form.
  • Check that it has a submit button.

Image - H37 Image Tag Alt Attribute

Description

  1. This method conforms to the WCAG rule H37.
  2. This rule identifies for the presence of alt text for all the image tag that is not an ancestor of anchor tag.Also checks for the presentational image through its width and height.

Procedure

  • Found all the image tags in the page.
  • Checked for the presence of alt text associated with it.

Alt Tags - H24 Image Map Alt Attribute

Description

  1. This method conforms to the WCAG rule H24.
  2. This rule identifies for the presence of alt attribute of the area element to provide text that describes the purpose of the image map areas.

Procedure

  • Get all area elements in an image map.
  • Check that the area element has an alt attribute.

Alt Tags - H35 Applet Tag Alt Attribute

Description

  1. This method conforms to the WCAG rule H35.
  2. This rule identifies the presence of text alternative for an applet by using the alt attribute to label an applet.

Procedure

  • View the source code of the applet element.
  • Check that the applet element contains an alt attribute with a text alternative for the applet.

Alt Tags - H53 Object Tag Alt Attribute

Description

  1. This method conforms to the WCAG rule H53.
  2. This rule identifies the presence of text alternative for content rendered using the object element.

Procedure

  • Get all object elements on the page.
  • Check that the body of each object element contains a text alternative for the object.

Alt Tags - H64 IFrame Tag Title Attribute

Description

  1. This method conforms to the WCAG rule H64.
  2. This rule identifies all the iframes or frames available in the page has title attribute, associated with the iframes used in the page.
  3. Note that the title attribute labels frames, and is different from the title element which labels documents.

Procedure

  • Get all frame and iframe elements on the page.
  • Check each frame and iframe element in the HTML or XHTML source code for the presence of a title attribute.

Anchor - H30 Opening New Windows

Description

  1. This method conforms to the WCAG rule H30.
  2. This rule identifies the presence of title as for all the links in the page which open in a new window.

Procedure

  • Get all anchor elements that have a title attribute.
  • Check that the title attribute together with the link text describes the purpose of the link.

Anchor - H30 Links Repeated

Description

  1. This method conforms to the WCAG rule H30.
  2. This method checks whether the label and description tags are sufficient to clarify the purpose of the link.

Anchor - H57 Unique IDs

Description

  1. This method conforms to the WCAG rule H75.
  2. This method checks whether anchor tags have unique ids.

Core - H46 Embed Tag

Description

  1. This method conforms to the WCAG rule H46.
  2. This rule identifies whether the alternative content is provided for the noembed element in a noembed element.

Core - H43

Description

  1. This method conforms to the WCAG rule H43.
  2. This rule identifies whether each data cell (in a data table) is associated with the appropriate headers.

Procedure

  • Get all data tables where any cell contains an id or header attribute.
  • Check that each id listed in the headers attribute of the data cell matches the id attribute of a cell that is used as a header element.
  • Check that the headers attribute of a data cell contains the id attribute of all headers associated with the data cell.
  • Check that all ids are unique (that is, no two elements in the page have the same id).
